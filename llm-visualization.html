Technical Notes

This tool uses simulated data to illustrate concepts. Attention weights are generated using deterministic pseudo-random functions seeded by sentence and head index. Embeddings use pre-computed 2D positions that approximate PCA projections of real word2vec/GloVe vectors. Generation uses a curated Markov-like vocabulary with temperature-scaled sampling.

Positional Encoding

Transformers process all tokens in parallel and have no built-in sense of order. Positional Encoding injects position information into the embeddings using sinusoidal functions at different frequencies.

Single Dimension Curve

Select a dimension pair to see its sinusoidal pattern across positions. Low-index dimensions oscillate rapidly; high-index dimensions change slowly.

Formula:
PE(pos, 2i)   = sin(pos / 100002i / dmodel)
PE(pos, 2i+1) = cos(pos / 100002i / dmodel)

Each dimension uses a different frequency. Dimension 0 completes many cycles; dimension 63 barely moves. The combination produces a unique fingerprint for every position up to ~10,000 tokens.

Scaled Dot-Product Attention — Step by Step

Walk through the exact matrix computation for a 3-token input (“cat sat mat”) with d_k = 4. See how Q, K, V projections produce the final attention output.

Attention(Q, K, V) = softmax(QKT / √d_k) · V
Queries (Q) ask “what am I looking for?”. Keys (K) answer “what do I represent?”. Values (V) carry the actual information passed forward. Dividing by √d_k prevents excessively large dot products that would make softmax gradients vanish.

LLM Training Pipeline

Modern LLMs are built through three distinct training phases. Click each phase to see its loss/reward curve and learn what happens at each stage.

Pre-training

Train on hundreds of billions of tokens from the web, books, and code. The model learns language structure, factual knowledge, and basic reasoning via next-token prediction. Loss typically starts at 10–12 and converges around 2–3.

RLHF (Reinforcement Learning from Human Feedback): Human raters compare pairs of model outputs and mark which is better. A reward model is trained on these preferences. PPO then optimizes the policy to maximize reward while a KL penalty prevents it from drifting too far from the SFT baseline. DPO achieves a similar result without a separate reward model.

Key insight: Each stage builds on the previous — pre-training provides general knowledge, SFT teaches the format, RLHF aligns values.

Feed-Forward Network (FFN) Neurons

Each Transformer layer contains a 2-layer FFN: FFN(x) = max(0, xW₁+b₁)W₂+b₂. The inner layer expands to 4× d_model. Hover over neurons to see their activation for each input token.

Hover a neuron bar to see its pre/post-activation value.

Architecture

  • Linear W₁: d_model → 4×d_model  (expand)
  • ReLU / GeLU: non-linear activation
  • Linear W₂: 4×d_model → d_model  (project)
  • Residual: output + input (skip connection)

Neuron Specialization

Early layers activate on syntactic features (punctuation, word position). Middle layers encode semantic concepts (animal-ness, royalty). Late layers encode task-specific patterns used for prediction. Each neuron acts as a “feature detector”.

Parameter count: In GPT-2 base (d_model=768), each FFN layer has 2×768×3072 = 4.7M parameters. With 12 layers, FFN accounts for ~56M of 117M total parameters — nearly half the model.

Post-LN (Original)

The original Transformer applies LayerNorm after the sublayer and residual add: LN(x + Sublayer(x)). This was found to make training unstable with large learning rates, requiring careful warm-up schedules.

Formula: LN(x) = γ ⋅ (x − μ) / √(σ² + ε) + β
where μ = mean(x), σ² = variance(x), γ and β are learned per-dimension. ε = 1e−5 prevents division by zero.
