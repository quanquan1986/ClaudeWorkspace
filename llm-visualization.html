How it works: Modern LLMs use Byte-Pair Encoding (BPE). Common sub-word sequences are merged into single tokens. This lets the model handle unseen words by composing them from known sub-tokens. GPT-4 uses ~100k tokens; common English words are typically 1 token each.

Word Embedding Space (2D PCA Projection)

Each dot is a word projected from high-dimensional embedding space. Words with similar meanings cluster together. Drag to pan, scroll to zoom.

Embeddings: Each word is mapped to a ~4096-dimensional vector. Relationships are preserved: king − man + woman ≈ queen. PCA reduces these to 2D for visualization while preserving as much variance as possible.

About This Tool

An interactive, self-contained HTML visualization of Large Language Model (LLM) internals. No server required — everything runs in your browser.

Features

  • Tokenizer — BPE-style token splitting
  • Attention — Multi-head attention heatmap
  • Transformer — Interactive architecture diagram
  • Embeddings — 2D PCA word vector visualization
  • Generation — Autoregressive sampling simulation

Key Concepts

  • Byte-Pair Encoding (BPE)
  • Scaled Dot-Product Attention
  • Multi-Head Attention
  • Positional Encoding
  • Feed-Forward Networks
  • Layer Normalization
  • Temperature / Top-p / Top-k sampling
  • Autoregressive generation

Technical Notes

This tool uses simulated data to illustrate concepts. Attention weights are generated using deterministic pseudo-random functions seeded by sentence and head index. Embeddings use pre-computed 2D positions that approximate PCA projections of real word2vec/GloVe vectors. Generation uses a curated Markov-like vocabulary with temperature-scaled sampling.
